## 场景

1. 成员变量很多，比如一个资源池 ResourcePoolConfig
2. 有参构造的参数很多，入参顺序可能填错
3. 那用 set 一个个注入？
	1. 用 set 可能会使对象在某个时刻处于无效状态，因为是逐步 set 的。（当然有的在乎这点，有的无所谓）
	2. 用 set 会使每个属性的注入是相对独立的，而 builder 的 build 方法可以提供一个统一的整合时机，比如可以做一些校验工作
